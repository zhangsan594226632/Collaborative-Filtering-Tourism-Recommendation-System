# 一、项目概述

## 1\.1 项目背景

随着国内文旅行业快速复苏，大众自助游、短途游需求持续增长。传统旅游信息分散在各类网页、短视频平台，游客无法集中查看景点介绍、游玩攻略、最新旅游资讯，选景困难、参考资料杂乱。同时传统线下旅游管理方式落后，景点分类混乱、资讯更新滞后、用户反馈无法收集、景点数据无法统计分析。

为解决以上问题，本次开发一套**基于SpringBoot\+Vue的旅游推荐系统**，采用前后端分离架构，面向普通游客提供景点浏览、景点检索、收藏景点、评论留言、资讯阅读等功能；面向管理员提供完整后台管理功能，实现用户管理、景点分类管理、景点信息管理、旅游资讯管理、轮播图管理、数据统计可视化等功能，实现旅游信息数字化、系统化、可视化管理。

## 1\.2 开发技术栈

### 后端技术（SpringBoot）

- 核心框架：SpringBoot 2\.7\.x，快速搭建稳定后端服务，统一 RESTful 接口风格

- 持久层框架：MyBatis\-Plus，简化 CRUD、分页查询、条件检索、批量操作

- 数据库：MySQL 8\.0，存储用户、景点、分类、资讯、收藏、评论、轮播图等全部业务数据

- 身份认证：JWT Token 实现无状态登录鉴权，区分游客、管理员角色权限

- 安全机制：用户密码 MD5 加密存储，接口权限拦截、未登录拦截

- 文件上传：本地文件存储方式，支持景点图片、资讯封面、轮播图上传展示

- 数据可视化：后端提供统计接口，支撑前台 ECharts 柱状图数据展示

- 工具依赖：Hutool 工具类，完成日期处理、参数校验、加密处理、分页封装

### 前端技术（Vue3）

- 核心框架：Vue3 \+ Vue Router \+ Pinia 状态管理

- UI 框架：Element Plus，搭建后台管理系统页面

- 网络请求：Axios 封装全局请求、响应拦截，自动携带 Token、处理登录过期

- 可视化图表：ECharts 实现景点收藏量柱状图统计展示

- 页面组件：轮播图、图片预览、分页组件、搜索筛选、评论列表、卡片展示

- 交互体验：收藏、取消收藏、发表评论、详情弹窗、条件筛选、模糊搜索

## 1\.3 系统角色

系统分为**普通游客用户**和**系统管理员**两类角色，权限完全隔离：

- **普通游客用户**：注册登录、浏览首页景点、查看旅游资讯、检索景点、收藏景点、发表评论、查看个人收藏列表

- **系统管理员**：管理所有用户、景点分类、景点信息、旅游资讯、首页轮播图、查看数据统计、管理用户评论

## 1\.4 系统整体功能概述

本系统分为**前台游客展示端**和**后台管理员管理端**。前台实现景点展示、资讯浏览、收藏、评论、检索功能；后台实现全量数据管理与可视化统计，覆盖旅游推荐平台完整业务需求。

## 2\.1 登录注册模块

用户可注册普通游客账号，输入账号、密码、姓名、手机号完成注册。登录时后端校验账号密码，签发 JWT Token，根据角色跳转前台或后台页面，未登录用户无法进入核心功能页面。

## 2\.2 前台游客功能模块

### 2\.2\.1 系统首页

- 顶部首页轮播图展示旅游宣传海报

- 展示热门旅游景点卡片，包含景点图片、景点名称、简介

- 展示最新旅游资讯推荐内容

### 2\.2\.2 景点信息浏览模块

- 支持**景点名称模糊搜索**

- 支持**景点分类筛选**，快速筛选不同类型景点

- 卡片式展示所有景点，分页加载

- 点击景点进入详情页面

### 2\.2\.3 景点详情模块

- 展示景点大图、景点名称、所属分类、详细介绍

- 展示景点收藏数量、浏览量

- 支持**收藏 / 取消收藏**景点

- 查看所有用户对此景点的评论内容

- 登录用户可发表新评论

### 2\.2\.4 旅游资讯模块

- 展示平台发布的旅游攻略、旅游新闻、出行资讯

- 支持资讯标题搜索

- 点击查看资讯详情图文内容

### 2\.2\.5 个人中心模块

- 查看、修改个人资料

- 我的收藏：展示用户所有收藏的景点，支持取消收藏

- 查看自己发布的所有景点评论

## 2\.3 后台管理员功能模块

### 2\.3\.1 后台数据统计首页

- ECharts 柱状图可视化统计：各景点收藏量统计

- 直观展示平台热门景点数据，为运营推荐提供数据支撑

### 2\.3\.2 轮播图管理

管理员可对首页轮播图进行新增、上传图片、修改、删除、排序、启用禁用操作，控制前台首页展示效果。

### 2\.3\.3 用户管理

管理所有注册游客账号，支持查询、新增、编辑、删除、重置用户密码，维护平台用户数据。

### 2\.3\.4 景点分类管理

维护景点分类字典，如自然风光、人文古迹、游乐园、乡村旅游等，用于前台景点筛选归类，支持增删改查。

### 2\.3\.5 景点信息管理

后台核心功能，管理员可录入、编辑、删除景点信息，包括景点名称、分类、景点图片、详细介绍、浏览量、收藏量等，统一管控前台展示的所有景点数据。

### 2\.3\.6 旅游资讯管理

管理员发布、编辑、删除旅游资讯，支持封面图上传、富文本内容编辑，更新前台旅游攻略与新闻内容。

### 2\.3\.7 评论管理

查看所有用户景点评论，支持删除违规评论，维护平台评论社区秩序。

# 三、数据库核心表设计

|字段|类型|说明|
|---|---|---|
|id|bigint|主键|
|username|varchar\(50\)|登录账号|

id、type\_name分类名称、sort排序

## 3\.3 景点表 travel\_spot

|字段|类型|说明|
|---|---|---|
|id|bigint|主键|
|name|varchar\(100\)|景点名称|
|type\_id|bigint|分类ID|
|img|varchar\(255\)|景点图片|
|intro|text|景点介绍|

## 3\.5 评论表 travel\_comment

user\_id、spot\_id、content评论内容、create\_time评论时间

## 3\.6 资讯表 travel\_news

title标题、img封面图、content资讯内容、publish\_time发布时间

## 3\.7 轮播图表 banner

img图片、sort排序、is\_show是否展示

# 四、核心业务流程

## 4\.1 用户登录鉴权流程

用户登录 → 后端校验账号密码 → 生成 JWT Token → 前端保存 Token → 所有请求自动携带 Token → 拦截器校验权限，未登录、权限不足直接拦截。

## 4\.2 景点浏览与收藏流程

用户前台浏览景点列表 → 搜索/筛选景点 → 进入景点详情 → 查看介绍与评论 → 收藏景点 → 个人中心查看我的收藏。

## 4\.3 后台管理流程

管理员登录后台 → 维护轮播图、分类、景点、资讯 → 审核管理用户评论 → 查看景点收藏数据统计 → 实现整个旅游平台内容管控。

# 五、项目部署方式

## 5\.1 后端部署

- 导入 MySQL 数据库脚本

- 修改 application\.yml 数据库配置、JWT 配置、文件路径

- Maven 打包：mvn clean package \-DskipTests

- java \-jar 启动项目

## 5\.2 前端部署

- npm install 安装依赖

- npm run build 打包 dist

- Nginx 托管静态文件，反向代理后端接口
